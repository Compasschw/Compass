Current State

How are CHW organizations currently getting session data into Pear Suite for claims submission?

Why We're Asking
We need to understand the existing data flow before proposing a new one. Are orgs doing manual entry, CSV uploads, or is there an integration path we can plug into?
Assumption to Validate
We suspect most CHW orgs rely on manual data entry or paper forms, creating a significant time gap between service delivery and claims submission.
3
Technical Architecture

What does your API surface look like — REST, GraphQL, webhooks, or file-based exchange?

Why This Matters
This determines our entire integration architecture and development timeline. Each approach has very different implications for real-time vs. batch processing.
We Can Support
  • REST API (preferred)
  • Webhook callbacks
  • GraphQL
  • Batch file exchange (SFTP/S3)
4
Data Schema

What data fields does Pear Suite require to submit a Medi-Cal claim for a CHW encounter?

Why This Matters
Compass needs to capture every required field during the CHW session. Missing fields = rejected claims = lost revenue for the org.
Our Goal
Map Pear Suite's required schema to Compass form fields so data flows cleanly without manual re-entry.
5
Security

How do you handle authentication for third-party integrations — API keys, OAuth, or org-scoped tokens?

Why This Matters
Auth model determines how we architect the integration layer. Per-org tokens vs. per-user OAuth have very different implementation patterns.
Compass Auth Stack
  • AWS Cognito (user auth)
  • JWT tokens
  • Role-based access control
  • Per-org data isolation
6
Compliance

What are your HIPAA requirements for integration partners — do you require a BAA and security review?

Why This Matters
PHI will flow between both systems. We need to know Pear Suite's compliance requirements so we can meet them and use the BAA to strengthen our investor story.
Our Current State
Compass audit flagged HIPAA at 1/10. Knowing Pear Suite's bar gives us a concrete compliance checklist to work toward.
7
Workflow

What does the claims lifecycle look like after submission — how would Compass get status updates?

Why This Matters
If Compass submits data but never knows if the claim paid out, CHW orgs lose visibility. Closing the loop is the entire value proposition.
Ideal State
CHW sees claim status in-app: submitted, accepted, denied, paid. Zero context-switching to another portal.
8
Development

Is there a sandbox or staging environment we could develop and test against?

Why This Matters
We need a non-production environment to build, test, and iterate on the integration. No sandbox = significantly longer development timeline.
What We Need
  • API endpoint(s) with test data
  • Test Medi-Cal member IDs
  • Mock claim responses
  • Rate limits / quotas
  • API documentation
9
Precedent

Are there other platforms integrating with Pear Suite today that we could reference as a model?

Why This Matters
If someone has already done this, we can learn from their patterns. This saves months of guesswork and shows Nick we're not trying to reinvent the wheel.
What We're Looking For
Integration patterns, API usage examples, common pitfalls, time-to-integration benchmarks from existing partners.
